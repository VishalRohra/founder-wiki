# Andrej Karpathy

Andrej Karpathy is a computer scientist who has worked at Stanford, OpenAI, and Tesla (where he led the Autopilot vision team). He is known for making deep learning concepts accessible through teaching and writing.

## Worldview

**Software is entering its third era.** Karpathy frames the history of software as three paradigms: Software 1.0 (human-written code), Software 2.0 (neural network weights learned from data, which he [described in 2017](https://karpathy.medium.com/software-2-0-a64152b37c35)), and Software 3.0 (LLMs programmed via natural language prompts) [1]. Each transition changes not just the tools but the nature of who can build software and how.

**Technology diffusion has reversed.** LLMs are the first technology that reaches everyday users before it reaches expert developers. Non-technical people can get useful output from natural language immediately, while developers are still building the tooling and paradigms. Karpathy describes this as [power going to the people](https://karpathy.bearblog.dev/power-to-the-people/) [1].

**We are building a new kind of computer.** The LLM is the CPU, English is the programming language, prompts plus tools are the applications. This reframes the entire software stack: instead of rigid UIs with predetermined flows, developers build tool ecosystems that agents navigate via natural language [1].

## Key Frameworks

- **Software 1.0 / 2.0 / 3.0**: A taxonomy for understanding how the programming interface has evolved from machine code to learned weights to natural language [1].
- **Vibe coding**: Karpathy coined or popularized this term for the practice of building software by describing what you want in natural language and letting AI generate the code. He [reflected on the practice](https://karpathy.bearblog.dev/vibe-coding-menugen) after building MenuGen this way [1].
